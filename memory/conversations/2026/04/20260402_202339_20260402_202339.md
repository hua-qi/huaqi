正是「**短路求值**」的功劳：

1. `x > 3` → `True`（5确实大于3）
2. 遇到 `or`，一看前面已经是 **True** 了，直接短路！
3. `y / 0` **根本不会执行**，所以不会报「除零错误」
4. 进入 `if` 分支，输出「执行了」

---

如果改成 `and` 就不一样了：
```python
if x > 3 and y / 0:  # 这里会报错！因为两边都要检查
```
