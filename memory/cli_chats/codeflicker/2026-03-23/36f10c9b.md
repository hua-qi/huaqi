**HR 服务台性能优化**

**项目背景**
HR 服务台（基于 Kaleido 低代码平台）核心业务流（如证明开具、落户、房补等）响应缓慢，证明开具服务后端流平均耗时达 **8.45 秒**，严重影响用户体验。

**个人工作内容**

1. **性能数据分析**：通过分析 Kaleido 平台的端到端性能埋点数据（资源初始化 P90 ≤1.5s、应用初始化 P90 ≤300ms、页面渲染 P90 ≤1s），定位各阶段性能卡点，沉淀出一套适用于 Kaleido 平台的性能数据处理脚本与分析方法论。

2. **后端流 JS→Java 改造**：将核心业务流（证明开具、落户、房补等主/子应用共计多条流）由 JS 开发重写为 Java 开发，消除 JS 执行效率低、循环操作冗余等瓶颈。

3. **回调流优化**：对业务流中的回调节点进行改造，将回调流统一迁移至 Java 实现，协同平台团队完成平台内置流的性能支持。

4. **参数传递优化**：减少流之间传递的冗余参数，规避因内存占用过大导致的 OOM 问题。

5. **子应用流同步优化**：在优化主应用 HTTPS 流的同时，同步优化证明、房补、北京工作居住证、北京社保等各子应用的关联子流。

**优化成果**
- 证明开具服务后端流优化完成后，测试环境下平均响应时间**提升约 50%**（由 ~8.45s 降至约 4s 量级）。
- 建立了 Kaleido 平台应用的性能分析体系，形成可复用的性能数据处理脚本与分析方案。
