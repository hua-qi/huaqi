## 核心方法论
提出了一种基于"目标对比"的任务执行循环，与 `ralph-loop` 的区别在于**显式分离三个阶段**：
1. **定义目标**（与设计、任务解耦）
2. **现状与目标对比**（发现 gap）
3. **按 gap 完善任务**（循环执行）

这种方式适用于三个粒度：宏观目标、单个 spec 任务、spec 内局部阶段。

## 实际示例
以 **edict（三省六部制）功能** 为案例，演示了如何分宏观、任务、局部三级使用该方法：

- **宏观级**：让 AI 生成目标/gap 分析/执行规划报告，发现 edict 缺少 workflow 级正式恢复能力，提出 L0/L1/L2 三层拆分方案
- **任务级**：针对具体 spec 任务，用同样的目标对比循环驱动代码实现
- **局部级**：在单个 spec 内的某个阶段，用目标对比保证局部正确性

## 核心价值
相比单一提示词循环，该方法通过**显式的目标定义与 gap 分析**，能更可靠地保证 AI 按照预期完成复杂多步骤任务，避免"堆命令"而不解决根本问题。
