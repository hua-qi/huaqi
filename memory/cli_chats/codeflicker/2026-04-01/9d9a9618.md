## 根本原因

### 问题1：`save_conversation` 依赖 `messages` 里有 `AIMessage`，但实际上没有

`save_conversation` 的 turns 构建逻辑：
```python
for msg in messages:
    if isinstance(msg, HumanMessage):
        user_msg = msg.content
    elif isinstance(msg, AIMessage) and user_msg:
        turns.append(...)
```

`generate_response` 返回的是 `{"messages": [response_msg]}`，**但当 LLM 触发了 tool_call 时，`response_msg.content` 是空字符串**（tool call 的 content 为空，内容在 `tool_calls` 字段里）。

图的路由是：`chat_response` → `tools` → `chat_response`（循环）→ `extract_user_info` → `save_conversation`。

此时 state 里的 `messages` 包含多个 `AIMessage`，但**第一个 AIMessage（触发 tool call 的那条）content 是空的**，只有最后一条才有真正的回复内容。`save_conversation` 却在遇到 `AIMessage` 时立刻配对，导致 `assistant_response` 是空字符串。

### 问题2：Chroma 向量库里几乎没有任何对话记录

向量库只有 1 条记录，还是日记类型，**没有任何历史对话被索引进去**。这是因为问题1导致 `save_conversation` 的 `if not response: return {}` 检查（response 为空字符串时直接返回），或 turns 为空时直接返回，`_index_to_chroma` 根本没有被调用到。

---

## 总结

- Huaqi 回复为空 → `save_conversation` 因 `response` 为空字符串提前返回，或 turns 配对时拿到空 content
- 向量库无内容 → `_index_to_chroma` 从未被成功调用
- 记忆检索无结果 → 向量库空 + 只扫今天 Markdown
