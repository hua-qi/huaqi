[2026-04-01 15:21:28] [user]: spec-dirven-development 中需要 Agents.md 吗
[2026-04-01 15:22:56] [assistant]: 在 Spec-Driven Development (SDD) 中，**不一定需要** `Agents.md`。

SDD 的核心文档通常包括：
- `requirements.md` - 需求规格
- `design.md` - 设计方案
- `tasks.md` - 实现任务拆分

`Agents.md` 是在 **AI-agent 驱动的 SDD 工作流**（如 Cline/Kiro 等工具的规范）中才会出现，用来定义哪些 agent 参与、各自的职责和协作方式。如果你的 SDD 流程不涉及多 agent 协作，则不需要。

你具体用的是哪套 SDD 工具或框架？
[2026-04-01 15:23:35] [user]: 现在有哪几套 SDD 工具或框架？
[2026-04-01 15:25:39] [assistant]: 目前主流的 SDD 工具/框架有以下几套：

| 工具 | 来源 | 特点 |
|------|------|------|
| **Amazon Kiro** | AWS | IDE 插件，内置 spec 工作流（requirements → design → tasks），有 Agents.md 概念 |
| **GitHub Spec Kit** | GitHub/Microsoft | 开源工具包，结构化 spec 文件驱动 Copilot |
| **Tessl Framework** | Tessl | 激进方式：spec 本身即源码，代码从 spec 直接生成（2025年底私测） |
| **LeanSpec** | 社区/开源 | 轻量级，从第一原则出发，适合个人/小团队 |
| **Claude Code SDD** | 社区实践 | 非官方，基于 CLAUDE.md + spec 文件配合 Claude Code 使用 |

其中 **Kiro** 是唯一明确引入 `agents.md` 的工具，用于定义 AI agent 的钩子（hooks）和行为规范。
