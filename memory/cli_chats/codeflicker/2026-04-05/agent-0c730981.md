### **5. 最近新增的 Transcript-Collector 设计文档**

**文件位置**：`/Users/lianzimeng/workspace/huaqi-growing/docs/designs/2026-05-04-codeflicker-transcript-collector.md`

**核心内容**：

**目标**：将 codeflicker 对话记录作为新数据源接入 huaqi，自动提炼技术决策和技能成长

**Codeflicker 文件路径**：
```
~/.codeflicker/threads/{thread_id}/.meta/transcripts/{timestamp}-{summary}.txt
```

**关键特点**：
- 每个 `.txt` 文件 = 一次完整会话
- **会话结束后一次性写入**（不持续追加）
- Header 中有完整时间范围（起止时间）

**采集方案**：选择 **方案 C（增量处理 + 会话结束汇总）**
- 监听 `on_created` 事件（watchdog）
- 不需要字节偏移追踪
- 不需要超时检测

**新增文件清单**（计划）：
| 文件 | 动作 | 说明 |
|------|------|------|
| `collectors/codeflicker_watcher.py` | 新增 | watchdog 监听 + 已处理记录 |
| `collectors/transcript_parser.py` | 新增 | 解析并清洗 transcript 文件 |
| `collectors/decision_detector.py` | 新增 | LLM 判断价值 + 提炼摘要 |
| `raw_signal/converters/cli_chat.py` | 新增 | 清洗内容→RawSignal 转换 |
| `scheduler/manager.py` | 微调 | daemon start 时启动 Watcher |

**数据流**：
```
~/.codeflicker/threads/*/. meta/transcripts/*.txt (on_created)
  ↓ watchdog
CodeflichterWatcher
  ↓
TranscriptParser (提取 header、user query、assistant 最终回复)
  ↓
DecisionDetector (LLM 轻量判断)
  ├─ 有价值 → 提炼结构化摘要
  └─ 无价值 → 丢弃
  ↓
RawSignal (source_type=AI_CHAT, metadata={tool_type: "codeflicker", ...})
  ↓
现有 DistillationPipeline → TelosEngine → TELOS 成长维度更新
```

**TELOS 维度映射**：
| 对话内容 | TELOS 维度 |
|---------|-----------|
| 学了新技术/库 | T (Technique) |
| 解决了难题 | T + O (Observation) |
| 架构决策 | T + S (Self，反映思维模式) |
| 有挫败感或成就感 | E (Emotion) |

## **现有存储机制总结**

### **三层存储架构**

#### **第一层：Markdown Memory（人类可读）**
- 位置：`~/.huaqi/memory/conversations/`
- 格式：Markdown with YAML frontmatter
- 触发：对话结束时调用 `memory_store.save_conversation()`
- 代码：`huaqi_src/cli/chat.py` 的 `chat_mode()` 和 `run_langgraph_chat()`

#### **第二层：SQLite Raw Signal（结构化数据）**
- 位置：`~/.huaqi/data/raw_signals.db`
- 表：`raw_signals`（标记 processed/distilled/vectorized 状态）
- 用途：支持复杂查询和信号处理流程
- 转换：Markdown → RawSignal → SQLite

#### **第三层：Git 版本管理（同步备份）**
- 位置：`~/.huaqi/.git` (本地) / GitHub (可选远程)
- 触发：chat_mode() 和 run_langgraph_chat() 退出时
- 代码：`huaqi_src/layers/data/git/auto_commit.py` 的 `GitAutoCommit`
- 功能：自动 `git add .` && `git commit` && `git push` (可选)

**Git 提交方法示例**：
```python
result = ctx._git.commit_memory_change(session_id)

```

**数据流向**：
```
对话内容 (MarkdownMemoryStore)
  ↓
RawSignal Pipeline (转换为信号)
  ↓
TELOS Engine (提炼维度信息)
  ↓
报告聚合引擎
├─ MorningBriefAgent
├─ DailyReportAgent
├─ WeeklyReportAgent
└─ QuarterlyReportAgent

```

**报告文件位置**：
- 晨间简报：`~/.huaqi/reports/daily/{YYYY-MM-DD}-morning.md`

**报告生成方式**：
