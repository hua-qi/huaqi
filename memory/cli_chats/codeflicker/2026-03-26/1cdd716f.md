[2026-03-26 19:17:35] [user]: mcp 协议详情
[2026-03-26 19:17:55] [assistant]: 基于 XP 的实现，MCP (Model Context Protocol) 协议包含以下核心概念：

## 1. 通信方式

| 方式 | 说明 |
|------|------|
| **Stdio** | 标准输入输出（XP 使用） |
| **HTTP/SSE** | 服务端推送（可选） |

XP 使用 `stdio_server`：
```python
from mcp.server.stdio import stdio_server

async with stdio_server() as (read_stream, write_stream):
    await app.run(read_stream, write_stream, ...)
```

## 2. 核心组件

| 组件 | 职责 |
|------|------|
| `Server` | MCP 服务端实例 |
| `Tool` | 工具定义（名称、描述、参数 Schema） |
| `TextContent` | 文本响应内容 |

## 3. 必需接口

### list_tools()
返回可用工具列表，每个工具包含：
```python
Tool(
    name="tool_name",           # 工具标识
    description="描述",          # LLM 看到的说明
    inputSchema={...}           # JSON Schema 参数定义
)
```

### call_tool(name, arguments)
执行工具调用，返回 `TextContent` 列表：
```python
@app.call_tool()
async def call_tool(name: str, arguments: dict) -> list[TextContent]:
    return [TextContent(type="text", text="结果")]
```

## 4. 数据流

```
┌─────────────┐     JSON-RPC      ┌─────────────┐
│   Client    │ ◄──────────────► │ MCP Server  │
│  (Claude)   │   stdio/HTTP     │  (xp server)│
└─────────────┘                  └─────────────┘
                                        │
                                 ┌──────┴──────┐
                                 ▼             ▼
                           search()      extract()
                           检索经验       提取经验
```

## 5. JSON-RPC 消息格式

```json
// 工具列表请求
{"jsonrpc": "2.0", "id": 1, "method": "tools/list"}

// 工具调用请求
{"jsonrpc": "2.0", "id": 2, "method": "tools/call", 
 "params": {"name": "search_best_practices", 
            "arguments": {"query": "react hooks"}}}
```

## 6. 初始化流程

1. Client 启动 Server 进程
2. Server 发送 `initialize` 握手
3. Client 发送 `initialized` 确认
4. Client 调用 `tools/list` 获取工具列表
5. 后续按需调用 `tools/call`

| 指标 | 说明 | 收集方式 |

| `rejected_count` | 被拒绝次数 | `record_feedback` |

## 自定义存储位置

通过环境变量 `XP_HOME` 设置：

```bash
# 临时设置（当前终端）
XP_HOME=/path/to/custom/dir xp stats

# 永久设置（~/.zshrc 或 ~/.bashrc）
export XP_HOME=/path/to/custom/dir
xp stats
```

## 数据迁移方法

经验数据和指标库可以手动迁移：

```bash
# 1. 创建新目录
mkdir -p /new/xp/path

# 2. 复制数据文件
cp ~/.xp/knowledge.json /new/xp/path/
cp ~/.xp/metrics.db /new/xp/path/

# 3. 可选：复制项目配置
cp ~/.xp/projects.json /new/xp/path/
cp ~/.xp/current_project /new/xp/path/

# 4. 使用新位置
XP_HOME=/new/xp/path xp stats
```

**注意**：目前不支持自动迁移命令，需要手动复制文件。迁移后建议运行 `xp stats` 验证数据完整性。

XP 的相关文件包括：
- `knowledge.json` - 经验数据
- `metrics.db` - 指标库
- `projects.json` - 项目配置
- `current_project` - 当前项目记录
- `watch_state.json` - 文件监听状态（可选）
