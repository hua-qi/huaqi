### Function 是什么

Function 是平台管理的一类原生资源（和卡片、结构化数据、字典、知识库等并列），本质是**一个可被调用的后端函数**，封装了具体的业务逻辑或数据查询能力。

---

### 数据卡片用 Function 作数据源时

```ts
{
  type: "function";
  functionId: string;           // 绑定的 Function 资源 ID
  transformRequest?: string;    // JS 函数体：处理入参 → 转成 Function 需要的格式
  transformResponse?: string;   // JS 函数体：处理返回值 → 转成卡片需要的数据格式
}
```

`transformRequest` 和 `transformResponse` 是为了做**数据适配**——因为 Function 的入参/出参格式不一定和卡片渲染所需的格式一致，需要中间转换。

---

### SQL vs Function 对比

| | SQL 模式 | Function 模式 |
|---|---------|------------|
| 数据来源 | 结构化数据视图/表 | 平台注册的 Function |
| 适用场景 | 数据直接来自数据库 | 数据需要业务逻辑处理 |
| 筛选项枚举 | 支持 `from: "data"` 去重取值 | 不支持 |
| 数据转换 | 不需要 | 可通过 `transformRequest/Response` |

PRD 中也特别提到：Function 场景下 DSL 的 `filters`、`charts` 等字段**基本保持不变**，只有 `dataSource` 部分不同。
