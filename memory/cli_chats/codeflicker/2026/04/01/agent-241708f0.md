**raw_signal/models.py** (113 行，核心数据模型):
- **SourceType**: JOURNAL, AI_CHAT, WECHAT, READING, AUDIO, VIDEO, IMAGE, CALENDAR, ABSENCE
- **RawSignal**: 原始信号数据结构
  - 字段: id, user_id, source_type, timestamp, ingested_at, content, metadata
  - 状态: processed, distilled, vectorized
- **RawSignalFilter**: 查询过滤器
- **各类 Metadata**: JournalMetadata, WechatMetadata, ReadingMetadata, AudioMetadata, AbsenceMetadata

**raw_signal/pipeline.py** (87 行，信号提炼编排):
- **DistillationPipeline**: 5 步信号提炼编排
  1. 分析信号 (Step1: analyze)
  2. 提取维度 (Step2: extract)
  3. 决策更新 (Step3: decide)
  4. 生成新内容 (Step4: generate)
  5. 判断成长事件 (Step5: judge)

#### 2.6.2 成长层 (`layers/growth/`)
```
growth/
└── telos/                      # TELOS 维度系统 (关键!)
    ├── __init__.py
    ├── context.py              # TELOS 上下文
    ├── dimensions/             # 维度定义
    │   └── __init__.py

    ├── growth_events.py        # GrowthEvent 成长事件
    ├── manager.py              # TelosManager (维度管理)
    ├── meta.py                 # 元数据
    └── models.py               # TelosDimension 数据模型 - 详解下
```

**telos/models.py** (141 行，TELOS 维度模型):
- **DimensionLayer**: CORE (核心) | MIDDLE (中间) | SURFACE (表面)
- **STANDARD_DIMENSIONS**: 9 个标准维度
  ```
  beliefs, models, narratives (核心层)
  goals, challenges, strategies (中间层)
  learned, people, shadows (表面层)
  ```
- **TelosDimension**: 维度类
  - 字段: name, layer, content, confidence, update_count, is_active, is_custom, history
  - 方法: to_markdown(), from_markdown()
- **HistoryEntry**: 更新历史记录

**telos/engine.py** (294 行，LLM 驱动的 5 步处理 - 详解):
```
Step1 (Analyze): 原始信号 → 维度识别 + 情绪强度
  ↓
Step2 (Extract): 相关信号聚类
  ↓
Step3 (Decide): 判断是否需要更新维度
  ↓
Step4 (Generate): 用新信号生成更新的维度内容
  ↓
Step5 (Judge Growth): 判断是否是成长事件
```

**关键类/方法**:
- **Step1Output**: dimensions, emotion, intensity, signal_strength, summary
- **Step3Output**: should_update, update_type, confidence, reason, suggested_content
- **Step4Output**: new_content, history_entry
- **Step5Output**: is_growth_event, narrative, title
- **TelosEngine.step1_analyze()**: 调用 LLM 分析信号
- **TelosEngine.step3_decide()**: 判断维度是否需更新
- **TelosEngine.step4_generate()**: 生成更新内容
- **TelosEngine.step5_judge_growth()**: 判断成长事件
- **TelosEngine.run_pipeline()**: 完整流程编排

class DistillationPipeline:
    """信号提炼管道：将 RawSignal 通过 5 步提炼写入 TELOS。

        """处理单条信号，返回处理结果摘要。
        
        流程:
        1. Step1: 分析信号→提取维度、情绪、强度
        2. 触发规则检查（信号数 >= threshold）
        3. 为每个维度执行 Step2-Step5
        4. 生成 GrowthEvent（如果是成长事件）

        step1_result = self._engine.step1_analyze(signal)
        self._signal_store.mark_processed(signal.id)
        
        # 2. 遍历提取的维度
        for dimension in step1_result.dimensions:
            # 3. 检查信号数是否达到阈值
            count = self._signal_store.count(...)
            if count < self._threshold:
                continue

            run_result = self._engine.run_pipeline(
                signal=signal,
                step1_result=step1_result,
                signal_summaries=summaries,
                days=self._days_window,
            )
            
            # 5. 保存成长事件
            if run_result["updated"] and run_result["growth_event"]:
                event = GrowthEvent(...)

```python
class TelosEngine:
    """LLM 驱动的信号→维度提炼引擎，包含 5 个处理步骤"""
    
    def step1_analyze(self, signal: RawSignal) -> Step1Output:
        """Step1: 分析原始信号
        
        输入: 原始信号内容
        输出: Step1Output(dimensions, emotion, intensity, signal_strength, summary)
        """
        prompt = _STEP1_PROMPT.format(
            telos_index=self._telos_index(),  # 当前维度索引
            active_dimensions="...",
            source_type=signal.source_type.value,
            content=signal.content,
        )
        response = self._llm.invoke(prompt)
        return Step1Output(**_parse_json(response.content))
    
    def step3_decide(self, dimension: str, signal_summaries: List[str], days: int) -> Step3Output:
        """Step3: 决策是否更新维度
        
        输入: 维度名、信号摘要列表、时间窗口
        输出: Step3Output(should_update, update_type, confidence, suggested_content)
        """
        dim = self._mgr.get(dimension)
        prompt = _STEP3_PROMPT.format(
            dimension=dimension,
            days=days,
            count=len(signal_summaries),
            signal_summaries="\\n".join(...),
            current_content=dim.content,
        )
        response = self._llm.invoke(prompt)
        return Step3Output(**_parse_json(response.content))
    
    def step4_generate(self, dimension: str, step3_result: Step3Output, 
                      signal_summaries: List[str]) -> Step4Output:
        """Step4: 生成更新的维度内容
        
        输入: 维度名、Step3 结果、信号摘要
        输出: Step4Output(new_content, history_entry)
        处理: 自动保存到 TelosManager，更新维度文件
        """
        dim = self._mgr.get(dimension)
        prompt = _STEP4_PROMPT.format(
            dimension=dimension,
            old_content=dim.content,
            signal_summaries="\\n".join(...),
            suggested_content=step3_result.suggested_content or "",
        )
        response = self._llm.invoke(prompt)
        result = Step4Output(**_parse_json(response.content))
        
        # 自动保存到磁盘
        entry = HistoryEntry(
            version=dim.update_count + 1,
            change=result.history_entry["change"],
            trigger=result.history_entry["trigger"],
            confidence=step3_result.confidence,
            updated_at=datetime.now(timezone.utc),
        )
        self._mgr.update(
            name=dimension,
            new_content=result.new_content,
            history_entry=entry,
        )
        return result
    
    def step5_judge_growth(self, dimension: str, layer: DimensionLayer,
                          step4_result: Step4Output) -> Step5Output:
        """Step5: 判断是否是成长事件
        
        判断标准:
        - 核心层变化 → 几乎总是成长事件
        - 中间层方向性转变 → 通常成长事件
        - 表面层日常积累 → 通常不是
        
        输出: Step5Output(is_growth_event, narrative, title)
        """
        dim = self._mgr.get(dimension)
        prompt = _STEP5_PROMPT.format(
            dimension=dimension,
            layer=layer.value,
            old_content=dim.history[-2].change if len(dim.history) >= 2 else "",
            new_content=step4_result.new_content,
            trigger=step4_result.history_entry.get("trigger", ""),
        )
        response = self._llm.invoke(prompt)
        return Step5Output(**_parse_json(response.content))
    
    def run_pipeline(self, signal: RawSignal, step1_result: Step1Output,
                    signal_summaries: List[str], days: int) -> Dict[str, Any]:
        """完整管道编排: Step1 → Step3 → Step4 → Step5
        
        对于 Step1 识别的每个维度:
        1. 调用 Step3 决策是否需要更新
        2. 如果需要，调用 Step4 生成新内容
        3. 调用 Step5 判断是否是成长事件
        
        返回: {"updated": bool, "growth_event": Optional[Step5Output]}
        """
```

## 八、旧代码与新分层架构的关系

```
旧代码布局 (huaqi_src/core/, reports/, learning/, pipeline/)
                    ↓
        包含业务逻辑、特定功能实现
                    ↓
新分层架构 (huaqi_src/layers/)
                    ↓
    数据流: raw_signal → (分析) → telos → (应用)
    ├── layers/data/raw_signal/       # 原始信号收集
    ├── layers/growth/telos/          # 维度提炼与管理
    └── layers/capabilities/          # 能力应用 (学习、报告等)
                    ↓
        旧代码通过 layers 暴露的接口访问数据
        新代码遵循清晰的数据流向
```

**迁移策略**: 
- 旧代码继续维护（兼容性）
- 新功能优先使用 layers 架构
- 逐步重构旧代码依赖 layers

## 十二、总结

**Huaqi** 是一个架构清晰、层次分明、设计优雅的个人 AI 同伴系统：

1. **核心创新**:
   - LangGraph 状态驱动的对话工作流
   - 5 步信号→维度提炼的 TELOS 引擎
   - 无需 Embedding 的混合检索
   - 人类可读的 Markdown 存储

2. **代码特点**:
   - 157 个 Python 文件，结构清晰
   - 旧代码与新分层架构并存
   - 强大的测试覆盖 (68 个测试文件)
   - 详尽的文档 (46 个 .md 文件)

3. **可扩展性**:
   - 插件化的数据源、处理器、平台
   - Provider Registry 模式支持报告扩展
   - 清晰的模块边界，易于添加功能

4. **质量保证**:
   - 单元测试、集成测试、E2E 测试框架
   - 代码组织规范、命名规范
   - 架构决策记录 (ADR)
   - 配置热重载支持

这是一个**值得深入学习和贡献**的开源项目！
