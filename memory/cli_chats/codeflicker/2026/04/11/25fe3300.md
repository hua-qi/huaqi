[2026-04-11 16:38:29] [assistant]: 核心问题是：完成信号是一个需要被**机器可靠解析**的标志，但当前它和普通的人类可读输出混在一起，没有结构化区分。

**优化思路：结构化输出，完成信号独立**

将最后的完成信号改为 JSON 格式单独一行输出，并加前缀让 AI 可以精准匹配：

```python
# 普通统计结果照常打印（给人看）
print(f"共 {total} 条记录")
print(f"分布：{dist}")
print(f"详细数据已保存到 {file_path}")

# 完成信号独立为 JSON，给 AI 解析
import json
print("__SKILL_RESULT__:" + json.dumps({
    "skill": "hr-data-query",
    "status": "DONE",
    "rows": total,
    "file": str(file_path)
}, ensure_ascii=False))
```

这样 data-analyst 只需要找 `__SKILL_RESULT__:` 开头的那一行，不需要全文扫描，也不会被 `dist` 内容干扰。

**几个关键设计原则：**

1. **前缀唯一**：`__SKILL_RESULT__:` 这样的前缀不会在普通输出中出现，匹配精准
2. **JSON 单行**：`json.dumps` 默认不换行，保证一行一个完整信号
3. **FAILED / PARTIAL 统一格式**：三种状态都用同一结构，data-analyst 用同一段解析代码处理

你觉得这个方向可以，还是有其他考虑？比如是否需要同时兼容当前的文本格式（渐进式迁移）？
