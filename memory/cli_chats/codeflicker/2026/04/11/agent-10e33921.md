### 1. Slash Command 定义和注册机制

**Slash command 的本质**是一个 **Command 对象**，包括两个主要类型：
- **Prompt-based commands** (skills)：执行 Markdown 内容的命令
- **Local commands**：执行 TypeScript/TSX 代码的命令

**核心解析文件**：`/Users/lianzimeng/workspace/claude-code/src/utils/slashCommandParsing.ts`

```typescript
parseSlashCommand(input: string) → ParsedSlashCommand
// 输入: "/search foo bar" 或 "/mcp:tool (MCP) arg1 arg2"  
// 输出: { commandName: 'search', args: 'foo bar', isMcp: false }
```

**命令注册流程**（在 `src/commands.ts` 中）：
```
getCommands(cwd) 
  ├─ getBundledSkills()          (内置 skills)
  ├─ getBuiltinPluginSkillCommands()  (built-in 插件 skills)
  ├─ getSkillDirCommands(cwd)   (文件系统 skills)
  ├─ getWorkflowCommands(cwd)   (工作流)
  ├─ getPluginCommands()         (插件命令)
  ├─ getPluginSkills()           (插件 skills)
  ├─ getDynamicSkills()          (动态发现的 skills)
  └─ COMMANDS()                  (硬编码命令)
```

**执行触发路径**：
```
User输入 "/commit msg"
  ↓
processUserInput() → parseSlashCommand()
  ↓
findCommand() 查找命令对象
  ↓
判断执行方式：
  ├─ context === 'fork' → executeForkedSlashCommand() (子agent执行)
  └─ context === 'inline' → processPromptSlashCommand() (内联执行)
```

### 2. Skill 加载和触发机制

**Skill 是 prompt-based command 的特殊形式，存储在文件系统中**

**核心加载文件**：`/Users/lianzimeng/workspace/claude-code/src/skills/loadSkillsDir.ts`

#### Skill 来源优先级：
```typescript
type LoadedFrom = 
  | 'bundled'             // 编译进二进制的 skills
  | 'skills'              // ~/.claude/skills/ 或 .claude/skills/
  | 'plugin'              // 插件提供的 skills
  | 'managed'             // 管理员/策略配置
  | 'commands_DEPRECATED' // 旧版 /commands/ 目录
  | 'mcp'                 // MCP 服务提供
```

#### SKILL.md 文件结构：
```yaml

---
name: "Display Name"           # 可选
description: "What this does"  # 必需或从首行提取
when-to-use: "Use when..."    # 可选
user-invocable: true          # 默认 true
model: "opus"                 # 可选，模型覆盖
context: fork                 # inline 或 fork
paths: ["src/**", "*.ts"]    # 可选，条件激活规则
allowed-tools: [bash, read]   # 可选，运行时工具
arguments: [arg1, arg2]       # 可选，参数名
effort: "high"                # 可选，effort 等级
---

# Markdown Content 
```

### 3. Slash Command 与 Skill 的区别

| 特性 | Slash Command | Skill |
|------|---------------|-------|
| 范围 | 所有 prompt 和 local command | 仅 prompt-based command |
| 存储 | 硬编码或 ~/.claude/skills/ | 文件系统 SKILL.md |
| 定义 | TypeScript/Markdown | Markdown |
| Model 可见 | 通过 SkillTool | 通过 SkillTool + prompt 提及 |
| 权限管理 | Tool 级别 | Skill 级别 |
| 条件激活 | 不支持 | 支持 (paths) |
| 执行环境 | inline 或 fork | inline 或 fork |

**关键区别**：
- **Skill 是 Slash Command 的子集**：所有 skill 都可作为 slash command，反之不然
- **Skill 能否作为 slash command 触发？** YES，Model 通过 SkillTool 调用，User 直接输入 `/skill-name` 也可

### 总结

1. **Slash Command** 是通过 `/command-name` 触发的指令，包括 skills 和 local commands
2. **Skill** 是一种特殊的 prompt-based slash command，存储在文件系统的 SKILL.md 中
3. **两者的关系**：Skill ⊂ Slash Command（所有 skill 都是 command，但不是所有 command 都是 skill）
4. **Skill 能作为 slash command**：YES，通过 SkillTool 由 Model 调用，或用户直接输入
5. **用户自定义位置**：`~/.claude/skills/` (全局) 或 `.claude/skills/` (项目级)，不要用 CLAUDE.md 或 .codeflicker
