## Vite 核心概念

**为什么快？**
- **开发环境**：利用浏览器原生 ES Modules，按需加载，不打包整个项目
- **生产环境**：使用 Rollup 进行打包，输出高度优化的静态资源

**vs Webpack**
| | Vite | Webpack |
|---|---|---|
| 冷启动 | 极快（毫秒级）| 慢（需打包全部） |
| HMR | 极快 | 随项目增大变慢 |
| 原理 | ESM + esbuild | Bundle-based |

## 关键特性

1. **即时服务器启动** - Dev server 启动不需要打包
2. **闪电般的 HMR** - 模块热替换只更新变更模块
3. **按需编译** - 只编译当前页面请求的文件
4. **开箱即用** - 支持 TS、JSX、CSS Modules、静态资源等

## 快速上手

```bash
npm create vite@latest my-app -- --template react-ts
cd my-app
npm install
npm run dev
```

## 配置文件 `vite.config.ts`

```ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: { port: 3000 },
  build: { outDir: 'dist' }
})
```

[2026-04-14 17:03:09] [assistant]: ## ESM（ES Modules）

ECMAScript 官方模块系统，即 `import/export` 语法：

```js
import { foo } from './foo.js'  // 静态导入
export const bar = 42
```

**关键**：浏览器原生支持 ESM，可以直接请求单个模块文件，不需要提前把所有文件打包成一个 bundle。Vite 开发时就是让浏览器直接加载 ESM 文件，省去了打包步骤。

---

## esbuild

用 **Go 语言**写的 JS/TS 构建工具，速度极快（比 tsc/babel 快 10~100 倍）。

Vite 用它来做：
- **依赖预构建**：把 `node_modules` 里的 CommonJS 包转成 ESM
- **TS/JSX 转译**：把 `.ts`、`.tsx` 文件转成浏览器能认识的 JS

```
你的源码 (.ts/.tsx)
      ↓ esbuild 转译（极快）
浏览器能执行的 ESM JS
```

---

## 两者关系

```
开发时：
  源码 → esbuild 转译 → ESM → 浏览器直接加载（无打包）

生产时：
  源码 → Rollup 打包 → 优化后的 bundle
```

简单说：**ESM 是标准，esbuild 是工具**，Vite 用 esbuild 的速度 + ESM 的按需加载，实现了极快的开发体验。
