```python
def validate_node(state):

        return "fix"   # 路由回修复节点
    return "write"     # 路由到写文件节点
```

### Console 面板的实现原理

**iframe 本身不能直接共享父页面的 console**，但可以通过以下方式桥接：

#### 方案：iframe 内部劫持 console，通过 postMessage 传出

**iframe 内注入的代码（在生成的组件代码之前插入）：**

```js
// 劫持 iframe 内的 console
['log', 'warn', 'error'].forEach(level => {
  const original = console[level]
  console[level] = (...args) => {
    original(...args)
    window.parent.postMessage({
      type: 'console',
      level,
      message: args.map(String).join(' ')
    }, '*')
  }
})

// 劫持未捕获的运行时错误
window.addEventListener('error', (e) => {
  window.parent.postMessage({
    type: 'console',
    level: 'error',
    message: e.message,
    stack: e.filename + ':' + e.lineno
  }, '*')
})
```

**父页面监听并展示：**

```js
window.addEventListener('message', (e) => {
  if (e.data?.type === 'console') {
    // 渲染到 Console 面板 UI
    appendToConsolePanel(e.data)
    // 同时可以把错误信息发给 Agent 触发修复
    if (e.data.level === 'error') {
      triggerAgentFix(e.data.message)
    }
  }
})
```

#### 效果
- 用户在 Console 面板中看到：组件的 `console.log`、运行时报错、React 的 Warning
- Agent 也能收到这些错误，触发自动修复循环
- 完全不依赖第三方，原生实现

### 第三节：iframe 中白名单依赖的注入方案

这是前端技术最关键的一环。目标：让 iframe 中的 JSX 代码能 `import` 内部组件库，且只允许白名单内的依赖。

---

**方案：Babel Standalone + Import Map**

流程如下：

```
后端返回 JSX 代码
    │
    ▼
前端构建 iframe HTML（动态拼接）
    │  包含：
    │  1. Import Map（声明白名单依赖的 URL 映射）
    │  2. Babel Standalone（在浏览器中编译 JSX → JS）
    │  3. console 劫持脚本
    │  4. 用户组件代码（通过 <script type="text/babel"> 注入）
    │
    ▼
iframe srcdoc = 拼好的 HTML 字符串
```

**iframe HTML 模板示意**：

```html
<!DOCTYPE html>
<html>
<head>
  <!-- 1. Import Map：白名单依赖映射 -->
  <script type="importmap">
  {
    "imports": {
      "react": "https://cdn/react.production.min.js",
      "react-dom/client": "https://cdn/react-dom-client.js",
      "@company/ui": "https://your-cdn/company-ui.esm.js",
      "dayjs": "https://cdn/dayjs.min.js"
    }
  }
  </script>
  <!-- 2. Babel Standalone -->
  <script src="https://cdn/babel.min.js"></script>
</head>
<body>
  <div id="root"></div>

  <!-- 3. console 劫持 -->
  <script>
    /* 前文所述的 postMessage 桥接代码 */
  </script>

  <!-- 4. 用户组件（JSX，由 Babel 编译） -->
  <script type="text/babel" data-type="module">
    import React from 'react'
    import { createRoot } from 'react-dom/client'
    import { Table, Button } from '@company/ui'

    /* === Agent 生成的组件代码 === */
    function UserComponent() { ... }
    /* ============================= */

    createRoot(document.getElementById('root')).render(<UserComponent />)
  </script>
</body>
</html>
```

**白名单强制执行**：
- Import Map 中只声明白名单包，未声明的包 `import` 时会直接报错（浏览器原生拦截）
- 这个错误会通过 postMessage 传回父页面，Agent 收到后进入修复节点

---

**内部组件库的 ESM 产物要求**：

内部组件库需要提供一个 **ESM 格式的构建产物**（`company-ui.esm.js`），可以通过 CDN 或本地服务提供，供 iframe 加载。这是这个方案的前提条件。

**两层机制**：

### 3. 如何保证业务逻辑没问题？

**Verification Agent**（`src/tools/AgentTool/built-in/verificationAgent.ts`）：

- 强制包含对抗性探针：并发、边界值、幂等性、孤立操作

**有，三层**：
1. **Stop Hooks**：可配置为任意 shell 命令 / LLM prompt / HTTP 回调，每轮结束自动执行
