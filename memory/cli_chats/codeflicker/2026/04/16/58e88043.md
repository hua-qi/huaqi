```tsx
<AppRenderer

  toolResourceUri="https://example.com/card.html?userId=123&theme=dark"
  sandbox={{ url: sandboxUrl }}
  onReadResource={async ({ uri }) => {
    // uri 包含完整的 URL 和参数
    const response = await fetch(uri);
```

原因：

| 对比维度 | SDK Client | HTTP 代理 |
|----------|-----------|-----------|
| **跨域问题** | 浏览器直接连 MCP Server 必然跨域 | ✅ 无跨域，CF-Agent 作为统一出口 |
| **协议兼容** | 只能连标准 MCP Server | ✅ 可适配任意协议（A2A、自定义 HTTP、gRPC） |
| **网络可达** | 要求 MCP Server 暴露公网 | ✅ SG-Agent 和 MCP Server 可在内网 |
| **安全管控** | 难做统一鉴权 | ✅ CF-Agent 集中处理 token、权限 |
| **多 Agent 支持** | 每个 Agent 需独立配置 | ✅ 统一接口 `/resource-proxy` 路由到不同 Agent |

**推荐架构：**

```
CF-Agent FE ──HTTP──→ CF-Agent ──协议适配层──→ 任意 Agent
                         │
                         ├── A2A ──→ SG-Agent ──MCP──→ MCP Server A
                         ├── HTTP ──→ Custom Agent B
                         └── MCP ──→ MCP Server C (直连)
```

**CF-Agent 统一接口设计：**

```tsx
// FE 调用（与后端协议无关）
<AppRenderer
  toolResourceUri="agent://sg-agent/ui://stargate/card/123"
  onReadResource={async ({ uri }) => {
    // uri 包含 agent 标识和目标资源
    const res = await fetch(`/api/agents/resources?uri=${encodeURIComponent(uri)}`);
    return res.json();
  }}
/>

// CF-Agent 路由
@app.get("/api/agents/resources")
async def read_resource(uri: str):  # agent://{agent_id}/{resource_uri}
    agent_id, resource_uri = parse_agent_uri(uri)
    
    if agent_id == "sg-agent":
        # A2A 协议转 MCP
        return await call_sg_agent_via_a2a(resource_uri)
    elif agent_id == "agent-b":
        # 直连 HTTP
        return await call_agent_b(resource_uri)
    # ... 更多 Agent
```

这样 AppRenderer 无需关心后端是 A2A、MCP 还是其他协议，CF-Agent 做协议适配即可。
