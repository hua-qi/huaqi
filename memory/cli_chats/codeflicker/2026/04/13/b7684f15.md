`manager.py` 的 `start()` 中，当没有 running loop 时（即 daemon `--foreground` 模式），用的是：


**`daemon=True` 意味着这个线程是守护线程**——当主线程（`system.py` 的 `while True: time.sleep(5)` 循环）退出时，守护线程会被立即杀死。但实际上主线程一直在运行，问题在于 `_run_scheduler` 的 asyncio 循环里：

```python

        await asyncio.sleep(3600)  # ← 每3600秒才 yield 一次
```

`AsyncIOScheduler` 需要 asyncio 事件循环持续运行才能触发任务，而 `await asyncio.sleep(3600)` 会让循环每小时才检查一次，导致所有任务都 miss 了。

正确做法应该是用 `BackgroundScheduler`（同步）或者让 asyncio loop 持续运行（`await asyncio.sleep(1)` 或者直接不用 sleep 的方式）。
