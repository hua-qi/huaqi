[2026-04-13 22:40:47] [assistant]: ## A2A Extensions 总结

### 是什么

Extension 是 A2A 的**协议扩展机制**，允许在不破坏核心标准的前提下，给协议叠加新数据、新方法、新状态。任何人都可以定义和发布扩展，**用 URI 唯一标识**。

---

### 四种扩展类型

| 类型 | 作用 | 举例 |
|---|---|---|
| **Data-only** | 在 AgentCard 里附加结构化信息，不影响请求流 | GDPR 合规数据 |
| **Profile** | 对核心消息做约束或用 metadata 扩展状态机子状态 | 要求所有 part 符合特定 schema；在 `working` 状态下用 metadata 标记 `generating-image` 子状态 |
| **Method（Extended Skill）** | 新增 RPC 方法 | `task-history` 扩展新增 `tasks/search` 方法 |
| **State Machine** | 给任务状态机新增状态或转换路径 | — |

---

### 声明方式（AgentCard）

在 `capabilities.extensions[]` 中声明，每条是一个 `AgentExtension` 对象：

```json
"capabilities": {
  "extensions": [
    {
      "uri": "https://example.com/ext/konami-code/v1",
      "description": "Provide cheat codes to unlock new fortunes",
      "required": false,
      "params": {
        "hints": ["When your sims need extra cash fast"]
      }
    }
  ]
}
```

- `uri` — 扩展唯一标识，建议含版本号（`/v1`）
- `required: true` — 客户端**必须**激活该扩展，否则 Agent 应拒绝请求；**Data-only 类型不应标 required**
- `params` — 扩展级别的配置参数

---

### 激活流程（运行时协商）

扩展默认**不激活**，通过 HTTP Header 协商：

```http
POST /agents/eightball HTTP/1.1
A2A-Extensions: https://example.com/ext/konami-code/v1

{ ...正常 JSON-RPC body...
  "params": {
    "message": { ... },
    "metadata": {
      "https://example.com/ext/konami-code/v1/code": "motherlode"
    }
  }
}
```

响应 Header 回显已激活的扩展：

```http
HTTP/1.1 200 OK
A2A-Extensions: https://example.com/ext/konami-code/v1
```

---

### 不能做什么（限制）

- **不能修改核心数据结构的字段定义**（新字段放 `metadata` map）
- **不能给 enum 类型加新值**（在 `metadata` 里标注附加语义）

---

### 治理与规范

- 官方扩展 URI 前缀：`https://a2a-protocol.org/extensions/`，仓库名前缀 `ext-`
- 实验性扩展前缀：`experimental-ext-`
- 规范文档**应托管在扩展 URI 本身**，推荐用 `w3id.org` 等永久 ID 服务防止链接失效
- **Breaking change 必须换新 URI**，Agent 不允许自动降级到旧版本
- 新增方法必须受同等鉴权保护，不得绕过安全控制

---

### 与本项目的关联

本项目的 `mcp_ui_resource` 扩展正是 **Profile Extension + Data Extension** 的实践：将 `resourceUri`/`toolResult`/`uiMetadata` 附加在 text part 的 JSON 里（等价于放 `metadata`），不新增 RPC 方法、不改变核心结构，完全符合 A2A 扩展规范。若要规范化，可将其定义为正式扩展并在 AgentCard 中声明：

```json

  {
    "uri": "https://your-org.com/ext/mcp-ui-resource/v1",
    "description": "在 A2A 响应中携带 MCP-UI 可渲染卡片资源",
    "required": false
  }
]
```

    for part in response_msg.parts:           # 遍历 parts，按类型路由
        if hasattr(part, "text"):
            parts.append({"kind": "text", "text": part.text})
        elif hasattr(part, "data"):
            if part.data.get("kind") == "mcp_ui_resource":
                parts.append(part.data)
    return parts
