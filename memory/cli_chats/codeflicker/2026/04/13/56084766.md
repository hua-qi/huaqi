- Present incrementally, validate as you go

- Don't edit or write code during brainstorming

## 设计方案（方案 A）

### 1. 漏斗区块：增加百分比

只在分母不为零时显示百分比，避免除零。

```
── 使用漏斗 ──────────────────────────────
  总任务数          42
  调用搜索          38   (90%)
  搜索命中          31   (82%)
  采纳经验          24   (77%)
  采纳后成功        20   (83%)
```

各行百分比分母：
| 行 | 分母 |
|----|------|
| 调用搜索 | 总任务数 |
| 搜索命中 | 调用搜索 |
| 采纳经验 | 搜索命中 |
| 采纳后成功 | 采纳经验 |

### 2. 经验库区块：全局展示（无需 project_id）

```
── 经验库 ────────────────────────────────
  总条数            15   (active=12  conflict=2  archived=1)
  平均 confidence   0.76
```

- `total` = experiences 表全量（或按 project 过滤后）
- `active / conflict / archived` = 按 status 分组
- `avg_confidence` = 仅统计 active 状态，避免被 archived 拉低失去参考意义

### 3. 数据来源

新增一条 SQL 查到经验库健康度，与现有漏斗 SQL 并列执行，**不修改 schema**。

```sql
SELECT
    COUNT(*) AS total,
    SUM(CASE WHEN status='active'           THEN 1 ELSE 0 END) AS active,
    SUM(CASE WHEN status='conflict_pending' THEN 1 ELSE 0 END) AS conflict,
    SUM(CASE WHEN status='archived'         THEN 1 ELSE 0 END) AS archived,
    AVG(CASE WHEN status='active' THEN confidence END)         AS avg_conf
FROM experiences
-- 可选 WHERE project_id = ?
```

### 4. 改动范围

仅修改 `xp_local_mcp/cli/stats.py`，涉及：
- `stats_cmd` 函数增加经验库查询
- 漏斗输出行追加百分比字符串（`f" ({pct}%)"` 格式，分母为零时省略）
- 追加经验库区块输出

无 schema 变更，无新依赖。

## 设计：`xp-local list`

### 命令签名

```
xp-local list [--query TEXT] [--project-id TEXT]
```

- `--query`：对 `title`、`problem`、`solution` 做 SQL `LIKE %keyword%` 过滤
- `--project-id`：不指定则自动检测当前 git 仓库，检测失败则显示全部项目

### 交互流程

```
共 15 条  (project: all)

  1. [repo-a] 部署时需要手动触发 cache 刷新          conf=0.82
  2. [repo-a] auth 模块修改后必须同步更新 token 表    conf=0.76
  3. [repo-b] CI 配置缺少 Z 会静默失败               conf=0.91
  ...

输入编号查看详情（q 退出）：_
```

选编号后展开单条详情（复用 review.py 已有的 `_show_experience`）：

```
  标题：部署时需要手动触发 cache 刷新
  问题：xxx...
  方案：xxx...
  教训：xxx
  置信：0.82   创建于：2026-03-01
  采纳次数：5   成功率：80%

按 Enter 返回列表
```

### 数据查询

两条 SQL：
1. 列表查询：从 `experiences` 按 active + 可选 project + 可选 LIKE 过滤，`ORDER BY confidence DESC`
2. 详情查询：单条 experience + LEFT JOIN `task_experience_usage` 统计采纳次数和成功率

### 改动范围

- 新建 `xp_local_mcp/cli/list_cmd.py`
- `cli/main.py` 注册新命令
- 复用 `review.py` 中的 `_show_experience`（可提取到共享模块，或直接复制）
- 新建 `tests/test_cli_list.py`
