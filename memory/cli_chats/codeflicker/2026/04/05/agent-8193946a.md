### 1. **Daemon 启动/管理实现** (`huaqi_src/cli/commands/system.py`)

#### 管理方式：**macOS LaunchAgent（plist）+ 前台模式运行**

**关键特点：**
- **无PID文件**：不使用传统的PID文件机制
- **使用macOS LaunchAgent**：通过plist配置文件管理daemon生命周期
  - Plist路径：`~/Library/LaunchAgents/com.huaqi.daemon.plist`
  - 使用`launchctl`命令加载/卸载
  
**启动流程：**

```
daemon install
  ↓
生成plist → 调用 launchctl load
  ↓
plist配置执行: huaqi daemon start --foreground
  ↓
Daemon以前台模式运行（KeepAlive: true自动重启）
```

**Plist配置关键字段：**
```xml
<key>RunAtLoad</key><true/>       # 开机自启
<key>KeepAlive</key><true/>       # 自动重启失败进程
<key>StandardOutPath</key>        # 日志输出
<key>ProgramArguments</key>       # 执行: huaqi daemon start --foreground
```

**支持的操作：**
- `daemon start [--foreground]` - 启动（前台/后台）
- `daemon stop` - 停止
- `daemon status` - 查看状态
- `daemon list` - 列出任务
- `daemon install` - 安装为开机自启（macOS only）
- `daemon uninstall` - 卸载开机自启

### 2. **调度器实现** (`huaqi_src/scheduler/manager.py`)

#### 核心特点：**基于APScheduler的AsyncIOScheduler**

**启动机制：**

```python
def start(self):
    if self.scheduler.running:
        return
    
    # 检查是否有现存的event loop
    try:
        loop = asyncio.get_running_loop()
    except RuntimeError:
        # 创建新的event loop + 后台线程
        def _start_async_scheduler():
            loop = asyncio.new_event_loop()
            asyncio.set_event_loop(loop)
            async def _run_scheduler():
                self.scheduler.start()
                while True:
                    await asyncio.sleep(3600)
            loop.run_until_complete(_run_scheduler())
        
        t = threading.Thread(target=_start_async_scheduler, daemon=True)
        t.start()
```

**关键设计：**
- 使用**守护线程**运行异步调度器
- SQLite持久化任务到：`~/.huaqi/data/scheduler.db`
- 支持Cron、Interval、Date三种触发器
- 事件监听机制（成功/失败/误触发）

### 6. **建议与发现**

**目前的设计问题：**
1. **Watcher未自动启动**：Daemon启动时不会启动CLIChatWatcher
2. **No Graceful Shutdown**：未实现watcher的优雅关闭机制
3. **No Integration Hook**：Scheduler中没有事件回调来启动watcher

**改进方向（如需）：**
- 在`daemon start`中添加watcher启动逻辑
- 在`daemon stop`中添加watcher停止逻辑
- 在Scheduler中添加integration hooks以供扩展

---

**总结：** Daemon通过macOS LaunchAgent管理，使用守护线程运行APScheduler；Watchdog Observer独立实现但未集成到daemon中，目前仅支持手动触发。
