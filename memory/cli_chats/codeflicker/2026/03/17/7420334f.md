## 天舟进销存系统 | 快手

**项目简介：** 天舟进销存系统是快手内部礼品/商品管理的综合平台，覆盖采购入库、库存管理、内部领用、售卖管理及财务管理全链路业务，前后端分离架构，迭代至五期。

**担任角色：** 前端开发（参与二期至五期全程开发，四/五期担任评审与技术方案负责人）

**核心工作内容：**

**一、复杂业务流程开发（领用申请单模块，三/四/五期主要负责人）**
- 开发内部领用申请单完整审批流程，支持项目/活动预算、一次性预算两套审批流，兼容财务BP审批节点、库管员审批节点等多审批人动态穿透逻辑
- 实现库管员审批节点下子库存分配功能：弹窗展示各子仓实时库存，支持多子仓出库数量拆分录入，联动后端占用/核销逻辑
- 对接预算系统接口，实现申请单提交/修改/审批状态全生命周期与预算系统的双向同步
- 支持撤回、驳回后单据重新编辑提交，处理一次性预算明细行段值必填的动态校验及弹窗提示
- 五期新增单据授权功能，扩充审批人权限列表查看、加签/转签/知会等协作操作

**二、财务管理模块（二/三期主要负责人，约4PD）**
- 实现领用入账确认页：列表筛选、批量导入/导出、确认入账一键操作
- 编辑抽屉中费用类别、费用明细、五段值（项目/产品/APP/地点/标签）均对接主数据接口实现模糊搜索，领用主体由入账部门自动带出
- 对接财务主数据 COA 接口，动态回显编码与名称

**三、数据看板模块（四期，约3PD）**
- 实现库存余额表、销售情况表报表页，支持重新计算与导出
- 库存余额表接入异步计算机制，通过操作记录状态轮询，利用 KIM 消息通知用户计算完成

**四、系统整体迭代与架构治理（二/三/四期，1～1.5PD/期）**
- 全局增加库存期间校验，涵盖采购/非采购接收、入库、退货、出库、售卖、领用各业务节点的时间合法性校验
- 商品管理、销售价目表新增导出功能；库存组织增加默认子仓库字段；SKU映射模块重构，支持1:N销售渠道编码映射，以全删全插策略保证数据一致性
- 四期主导评审与技术方案设计（1.5PD）

**技术亮点：**
- 可编辑表格组件高复用设计，覆盖子仓出库拆分、入账明细编辑等多处场景
- 基于模型驱动视图思路实现报表管理页，逻辑复用最大化
- 异步计算任务状态管理（计算中/已完成）结合 KIM 消息提醒，优化用户等待体验

**技术栈：** Vue / React（Kaleido 平台）、前后端分离、BPF 审批流、财务主数据接口对接
