## 快手内部部署方案

### 容器部署平台

快手内部有两个可选平台：

**1. KD Serverless（推荐，适合 HTTP 服务）**
> 但仅支持 **Node.js**，我们的 Python 项目不适用。

**2. Kubernetes（通用，推荐用于 Python 服务）**
- 标准 K8s Deployment + Service 部署
- 需将 Kess Agent 作为 **Sidecar** 容器运行（用于访问内部密钥服务 KcStore）

```yaml
# k8s Deployment 示例骨架
containers:
  - name: code-wisdom-mcp
    image: your-registry/code-wisdom-mcp:latest
    env:
      - name: MCP_TRANSPORT
        value: "http"
      - name: GEMINI_API_KEY
        valueFrom:
          secretKeyRef:
            name: code-wisdom-secrets
            key: gemini-api-key
  - name: kess-agent          # 内部密钥代理 Sidecar
    image: internal/kess-agent:latest
```

### 鉴权方案

快手内部有两种鉴权体系，取决于调用方是**人**还是**机器**：

| 场景 | 方案 | 说明 |
|------|------|------|
| **机器对机器**（agent 调用 MCP） | **Bearer Token（AK/SK）** | 通过 KcStore 的 `AKSKClient` 下发，在请求头携带 |
| **人工访问/管理页面** | **CAS SSO** | 内部 SSO：`sso.corp.kuaishou.com`，依赖 `infra-cas-sdk` |

**对于我们的 MCP Server（agent 调用），推荐 AK/SK 方案：**

```python
# 用 KcStore 替换硬编码 token
from kcstore.client import AKSKClient

client = AKSKClient(key_name='kcStore.aksk.code-wisdom')
ak = client.get_ak()
sk = client.get_sk()
```

密钥管理入口：`https://kc.corp.kuaishou.com/keyName/apply`
