<div>## Kaleido 页面性能现状收集思路

</div>
<div>
</div>




- 操作：“大刷”后资源加载



- 查看页面加载链路及分布占比；

- 给出 performance 加载链路和饼图占比；

- 给出治理意见；



- 对比 kaleido 线上环境和测试环境，查看是否有差异，给出标准；



- 询问峰哥

- 若测试环境和线上没有差异，则优先“证明”系统；

- 若有差异，则优先“社保”系统



- 性能优化域划分



- 三个阶段



- 开始 —&gt 框架加载完成【平台类】

- 框架加载完成 —&gt 导航加载完毕【平台类，可能没有】

- 导航加载完毕 —&gt 页面加载完毕【搭建类】





- 使用 Mac 和 windows 两种操作系统；





<div>
</div>
<div>	1. 【拿不到生产环境的时间节点日志】结合平台侧给出的时间节点和时间计算逻辑，对比同一系统同一页面下渲染时间细节差异，作为最终报告的参考值</div>
<div>	2. 收集同一系统下不同页面的三个时间阶段的值。（多次取平均）</div>
<div>	3. 结合步骤 1. 给出的差异值做报告；</div>
<div>	4. 针对上述提到的 搭建类 阶段的下述类别做分析、给建议：</div>
<div>		4.1. 请求时间（大致已有）</div>
<div>		4.2. 长任务，仔细分析</div>

<div>    权限相关</div>
<div>
</div>




- 编辑时可定义用户组管理员和用户组成员



- 用户组管理员不等于用户组，对于同时是管理员和用户的成员需要加两次

- 不同分支下用户组成员不同步



- 原因如下：



- preview 和 test 环境下具有分支功能，不同分支在权限表通过一个字段做区分，在获取时不同步

- PS： 进入 sendBox 和 online 需要将开发分支合并到主分支进行操作







- 运行时下的四个环境定义的用户组成员不同步



- 四个环境各对应一张权限表
